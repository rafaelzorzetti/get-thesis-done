function extractBibKeys(bibContent) {

  let match;

  }
  return keys;
}

Add the following functions to gtd-tools.js, placed between the existing `cmdFrameworkUpdate` function and the CLI router section:

**1. extractBibEntries(bibContent)** -- Extract complete BibTeX entries (not just keys) for import/merge.
- Uses `/@(\w+)\{([^,\s]+),/g` to find entry starts
- Tracks balanced braces to find each entry's closing `}`
- Returns array of `{ type, key, text }` objects
- Skips entries whose type starts with "comment" or "preamble" or "string" (BibTeX special entries)

**2. appendBibEntry(bibPath, entryText)** -- Safely append a BibTeX entry to references.bib.
- Reads existing file content
- Ensures a blank line separator before the new entry
- Writes back with UTF-8 encoding
- If file does not exist, creates it with a header comment: `% References -- managed by get-thesis-done`
- Validates by re-reading and extracting keys to confirm entry was added

**3. cmdImportBib(cwd, filePath, raw)** -- REF-01: Import entries from external .bib file.
- Accepts `--file path.bib` argument
- Reads source .bib file via safeReadFile, error if not found
- Reads existing references.bib and extracts existing keys via extractBibKeys
- Extracts entries from source via extractBibEntries
- For each entry: if key already in existing keys, skip (add to skipped list); if unique, append via appendBibEntry (add to imported list)
- Output JSON: `{ imported: number, skipped: number, imported_keys: string[], skipped_keys: string[], target: 'src/references.bib' }`
- Raw output: human-readable summary ("Imported N entries, skipped M duplicates")

**4. cmdValidateRefs(cwd, raw)** -- REF-04: Cross-chapter citation validation.
- Reads src/references.bib, extracts valid keys
- Scans TWO locations:
  - `src/chapters/*.tex` (approved chapters)
  - `.planning/chapters/*/*.tex` (drafts -- only the latest: prefer DRAFT-r2.tex > DRAFT-r1.tex > DRAFT.tex pattern, using the highest numbered file matching `*DRAFT*.tex`)
- For each .tex file found: run validateCitations(content, validKeys)
- Aggregate results: allCitedKeys (Set union), missingFromBib (cited but not in .bib), orphanedInBib (in .bib but never cited)
- Output JSON: `{ total_bib_entries, total_cited_keys, missing_from_bib: [], orphaned_in_bib: [], per_chapter: [{ file, source: 'approved'|'draft', valid_count, invalid_count, invalid_keys: [] }] }`
- Raw output: formatted summary with per-chapter breakdown and aggregate stats
- Use fs.readdirSync with try/catch for both directories (either may not exist yet)

**5. cmdPdfRefs(cwd, raw)** -- REF-05: Cross-reference cited keys with PDFs.
- Reads src/references.bib, extracts valid keys
- Lists PDF files in `src/references/` directory (case-insensitive .pdf extension)
- Matches: strip .pdf extension from filename, compare lowercase against lowercase bib key
- Output JSON: `{ total_entries, with_pdf, without_pdf, with_pdf_keys: [], without_pdf_keys: [] }`
- Raw output: human-readable table of keys with/without PDFs
- If `src/references/` directory does not exist, report 0 PDFs with a note

**6. Wire into CLI router** -- Add three new cases to the switch statement:
- `case 'import-bib':` reads `--file` arg, calls cmdImportBib
- `case 'validate-refs':` calls cmdValidateRefs (no args needed)
- `case 'pdf-refs':` calls cmdPdfRefs (no args needed)
- Update the help/usage string in the error message and file header comment to include all new commands

Do NOT make main() async yet -- that happens in Task 2. These three commands are synchronous.

**Part A: Add async functions to gtd-tools.js** (place after cmdPdfRefs, before CLI router)

**1. checkSystemDep(command)** -- Check if a system command is available.
- Uses `execSync('which ' + command, { stdio: 'pipe' })` in try/catch
- Returns boolean
- Used by pdf-meta to detect pdfinfo/pdftotext availability

**2. extractDOIFromPdf(pdfPath)** -- REF-03 helper: graduated PDF metadata extraction.
- Requires pdfPath as argument
- Strategy 1: Run `pdfinfo "${pdfPath}"` via execSync, scan output lines for DOI regex `/\b(10\.\d{4,9}\/[-._;()/:a-zA-Z0-9]+)\b/`
- Strategy 2: If strategy 1 fails, run `pdftotext -f 1 -l 2 "${pdfPath}" -` via execSync, scan text for same DOI regex
- If DOI found, return `{ doi, source: 'metadata'|'text' }`
- If no DOI found, return null
- Wrap each execSync in try/catch (command may fail or not be installed)
- Quote pdfPath in shell command to handle spaces

**3. constructMinimalBibEntry(pdfPath)** -- Fallback when no DOI found in PDF.
- Run `pdfinfo "${pdfPath}"` to get Title and Author fields
- Generate a citation key from first author lastname + year (or 'unknown_YYYY' if no author)
- Construct a `@misc{key, author={...}, title={...}, year={...}, note={PDF imported by get-thesis-done -- verify and complete this entry}}` entry
- Return the BibTeX string
- If pdfinfo is not available, construct an entry with just the filename as title and a TODO note

**4. async cmdFetchDoi(cwd, doi, raw)** -- REF-02: Fetch BibTeX from DOI via Crossref.
- Normalize DOI: strip `https://doi.org/`, `http://doi.org/`, `doi:` prefixes, trim whitespace
- Validate format: must match `/^10\.\d{4,9}\//`; error if not
- Fetch `https://doi.org/${encodeURIComponent(normalizedDoi)}` with headers:
  - `Accept: application/x-bibtex`
  - `User-Agent: get-thesis-done/1.0 (mailto:user@example.com)`
  - `redirect: 'follow'` (explicit for clarity)
- Check response.ok; error with HTTP status if not
- Check content-type header: if it does NOT contain 'bibtex' and does NOT contain 'text/plain', error with "DOI may not support BibTeX content negotiation" message
- Check if response body starts with `<!DOCTYPE` or `<html` (HTML fallback detection); error if so
- Get bibtex text from response
- Call appendBibEntry to add to src/references.bib
- Extract the key from the fetched BibTeX using extractBibKeys for reporting
- Output JSON: `{ doi, key, bibtex, appended_to: 'src/references.bib' }`
- Raw output: "Fetched and added:\n" + bibtex

**5. async cmdPdfMeta(cwd, filePath, raw)** -- REF-03: Extract metadata from PDF.
- Validate file exists via fs.existsSync; error if not
- Validate file ends with .pdf (case-insensitive); error if not
- Check for pdfinfo/pdftotext availability via checkSystemDep:
  - If neither available, error with platform-specific installation instructions (apt install poppler-utils / brew install poppler)
- Call extractDOIFromPdf(filePath)
- If DOI found:
  - Call cmdFetchDoi internally (reuse the fetch logic, but pass the result back instead of outputting directly)
  - Actually, better approach: extract the fetch logic. Call `fetchBibtexFromDoi(doi)` as a separate async helper that returns the BibTeX string, then appendBibEntry. Both cmdFetchDoi and cmdPdfMeta can call fetchBibtexFromDoi.
  - Output JSON: `{ pdf: filePath, doi, doi_source, bibtex, appended_to: 'src/references.bib' }`
  - Raw: "Found DOI {doi} in PDF {source}. Fetched and added:\n" + bibtex
- If no DOI found:
  - Call constructMinimalBibEntry(filePath)
  - Call appendBibEntry to add the minimal entry
  - Output JSON: `{ pdf: filePath, doi: null, minimal_entry: true, bibtex, appended_to: 'src/references.bib', warning: 'No DOI found. Minimal entry created -- please verify and complete.' }`
  - Raw: "No DOI found in PDF. Created minimal entry (please verify):\n" + bibtex
