<!-- CLI commands available from Plan 01 (all in gtd-tools.js) -->

import-bib --file path.bib [--raw]
  Input: path to external .bib file
  Output JSON: { imported, skipped, imported_keys, skipped_keys, target }
  Raw: "Imported N entries, skipped M duplicates"

fetch-doi --doi 10.xxxx/... [--raw]
  Input: DOI string (with or without https://doi.org/ prefix)
  Output JSON: { doi, key, bibtex, appended_to }
  Raw: "Fetched and added:\n" + bibtex

pdf-meta --file path.pdf [--raw]
  Input: path to PDF file
  Output JSON: { pdf, doi, doi_source, bibtex, appended_to, [warning] }
  Raw: "Found DOI ... Fetched and added:\n" + bibtex  OR  "No DOI found. Created minimal entry:\n" + bibtex

validate-refs [--raw]
  Input: none (scans all chapters automatically)
  Output JSON: { total_bib_entries, total_cited_keys, missing_from_bib, orphaned_in_bib, per_chapter }
  Raw: formatted summary with per-chapter breakdown

pdf-refs [--raw]
  Input: none (scans references.bib + src/references/)
  Output JSON: { total_entries, with_pdf, without_pdf, with_pdf_keys, without_pdf_keys }
  Raw: table of keys with/without PDFs

**<purpose>** section:
Orchestrate reference addition from multiple sources (.bib file, DOI, PDF) into the thesis bibliography. Auto-detect input mode, delegate to the appropriate CLI command, then run cross-chapter validation and PDF cross-reference checks. Present all results to the user with actionable recommendations.

**<core_principle>** section:
CLI commands do the mechanical work (import, fetch, extract, validate). This workflow orchestrates the sequence and presents results. For simple imports (bib file, DOI), no agent is needed -- the CLI output is sufficient. For complex cases (PDF with no DOI, validation failures), the reference-manager agent is spawned for judgment calls.

**<process>** section with the following steps:

**Step 1: Receive and Detect Input Mode**
- Receive argument from user via `/gtd:add-reference <source>`
- Auto-detect mode using these rules (test in order):
  1. If argument ends with `.bib` (case-insensitive) -> MODE: import-bib
  2. If argument matches `/^10\.\d{4,9}\//` -> MODE: fetch-doi
  3. If argument contains `doi.org/` -> MODE: fetch-doi (extract DOI from URL: strip `https://doi.org/` prefix)
  4. If argument ends with `.pdf` (case-insensitive) -> MODE: pdf-meta
  5. Otherwise -> Ask user: "Could not auto-detect input type. Please specify: Is this a DOI, a .bib file path, or a PDF file path?"
- Validate that the file exists (for .bib and .pdf modes) using `test -f`
- Report detected mode to user: "Detected mode: {mode} for input: {source}"

**Step 2: Execute Import/Fetch/Extract**
- Based on detected mode, run the corresponding CLI command:

For import-bib mode:
```bash
node get-thesis-done/bin/gtd-tools.js import-bib --file "{source}" --raw
```
- Parse output. Report imported count and skipped duplicates to user.
- If any entries were skipped due to key collision, list the skipped keys and ask if user wants to review them.

For fetch-doi mode:
```bash
node get-thesis-done/bin/gtd-tools.js fetch-doi --doi "{doi}" --raw
```
- Parse output. Show the fetched BibTeX entry to user.
- If fetch fails (network error, invalid DOI, content negotiation failure), report the error clearly and suggest: (1) check DOI format, (2) try accessing https://doi.org/{doi} in browser, (3) manually add the entry.

For pdf-meta mode:
```bash
node get-thesis-done/bin/gtd-tools.js pdf-meta --file "{source}" --raw
```
- Parse output. If DOI was found and fetched, show the BibTeX entry.
- If minimal entry was created (no DOI found), show the entry with a clear warning: "This is a minimal entry extracted from PDF metadata. Please review and complete the following fields: journal/booktitle, volume, pages, doi."
- If poppler-utils is not installed, report the error with installation instructions and suggest providing the DOI directly instead.

**Step 3: Post-Import Validation**
After any successful import/fetch/extract, automatically run validation:

```bash
node get-thesis-done/bin/gtd-tools.js validate-refs --raw
```

Present results to user:
- If 0 missing citations: "All citations across all chapters resolve to references.bib entries."
- If missing citations exist: List them per chapter with a suggestion to add the missing references.

```bash
node get-thesis-done/bin/gtd-tools.js pdf-refs --raw
```

Present PDF availability:
- "{N} of {M} referenced works have corresponding PDFs in src/references/"
- List keys without PDFs (if any) as an informational note, not a blocker.

**Step 4: Commit Changes**
If references.bib was modified:
```bash
git add src/references.bib
git commit -m "refs: add reference(s) via /gtd:add-reference"
```
Report the commit to user.

**Step 5: Summary**
Present a final summary:
- Mode used (import-bib / fetch-doi / pdf-meta)
- Entries added (count and keys)
- Validation status (all clear / N missing citations)
- PDF availability (N/M with PDFs)
- Next recommended action (if validation found issues, suggest adding missing references)

**Important workflow constraints:**
- The workflow does NOT spawn the reference-manager agent for simple operations. Agent spawn is reserved for when the user explicitly asks for a full validation review (future enhancement).
- The workflow handles one source at a time. For batch DOI import, the user runs `/gtd:add-reference` multiple times (the workflow is fast enough for sequential use).
- All error messages use thesis-native terminology.
- The workflow never modifies chapter .tex files.
