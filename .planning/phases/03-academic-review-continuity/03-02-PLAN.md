**Part A: Create get-thesis-done/workflows/review-chapter.md** (adapted from get-writing-done/workflows/review-chapter.md, 517 lines)

This workflow orchestrates the adversarial review cycle for a single chapter.

Terminology swaps (apply throughout):
- BIBLE.md -> FRAMEWORK.md, OUTLINE.md -> STRUCTURE.md, DRAFT.md -> DRAFT.tex
- gwd-tools.js -> gtd-tools.js, get-writing-done -> get-thesis-done
- /gwd:* -> /gtd:*, DRAFT-r1.md -> DRAFT-r1.tex, DRAFT-r2.md -> DRAFT-r2.tex

Workflow steps (adapt from GWD, preserve gates):

1. **Initialize** -- Receive chapter number. Validate prerequisites using `gtd-tools.js context --chapter N`. Check chapter has a draft (DRAFT.tex exists). Set `revision_count = 0`, `max_revisions = 2`, `current_draft_path = NN-01-DRAFT.tex`.

2. **Spawn reviewer agent** -- Use Task() to spawn reviewer with fresh context:
```
Task(
  subagent_type="general-purpose",
  prompt="
    First, read ~/.claude/agents/reviewer.md for your role and instructions.

    <chapter>$CHAPTER_NUMBER</chapter>
    <chapter_dir>$CHAPTER_DIR</chapter_dir>
    <draft_path>$CHAPTER_DIR/$PADDED-01-DRAFT.tex</draft_path>
    <plan_path>$CHAPTER_DIR/$PADDED-01-PLAN.md</plan_path>
    <output_path>$CHAPTER_DIR/$PADDED-01-REVIEW.md</output_path>
  ",
  description="Review Chapter $CHAPTER_NUMBER"
)
```

3. **Present findings** -- Read REVIEW.md, display score and failed checks to user. Show category breakdown (citation_validity, methodological_rigor, argumentative_coherence, formatting_norms).

4. **Decision gate** -- If status is "passed": proceed to finalization. If status is "needs_revision" and revision_count < max_revisions: offer revision. If revision_count >= max_revisions OR status is "failed": force final decision.

5. **Revision cycle** (if chosen) -- Increment revision_count. Spawn writer agent in Wave 2 revision mode with `<revision>` block containing failed_checks from REVIEW.md. Writer produces DRAFT-rN.tex. Update current_draft_path. Return to step 2 for re-review (re_review: true in REVIEW.md).

6. **Forced decision** (after 2 revisions) -- Present final review to user. User must choose: "approve" (accept current state, proceed to continuity loop) or "defer" (leave chapter for later manual revision, do NOT run continuity loop).

7. **Finalization** -- If approved: copy final draft to src/chapters/NN-slug.tex (same pattern as write-chapter finalize step). Git commit review artifacts. Set chapter status. Return status "approved" to caller.

Key differences from GWD:
- Draft paths use .tex extension throughout
- 4 categories instead of 6 in review presentation
- Revision drafts are DRAFT-r1.tex, DRAFT-r2.tex (not .md)
- Writer revision uses LaTeX-native output
- Direct git add/commit (not CLI wrapper)
