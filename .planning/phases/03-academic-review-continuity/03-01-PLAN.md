**Replace 6 GWD categories with 4 thesis categories:**

Category 1 - Citation Validity (3-5 subchecks):
- Run `gtd-tools.js validate-citations --chapter N` for automated pass/fail on all \cite{} keys
- Check every key argument has supporting \cite{} or \textcite{}
- Check citation commands match intent: \textcite{} for narrative, \cite{} for parenthetical, \autocite{} for ABNT
- Check citations are placed at substantive claims, not piled at paragraph ends
- Cross-reference against PLAN.md planned_citations -- flag missing planned citations

Category 2 - Methodological Rigor (3-5 subchecks):
- Methods in chapter consistent with FRAMEWORK.md Methodological Commitments table
- Claims supported by appropriate evidence type (empirical claims need data, theoretical claims need literature)
- Analytical approach matches STRUCTURE.md Methodological Arc position for this chapter
- Research limitations acknowledged where relevant (honest scope declaration, not weak hedging)

Category 3 - Argumentative Coherence (3-5 subchecks):
- Chapter thesis stated and supported across sections
- Arguments build logically -- no gaps, no contradictions within the chapter
- FRAMEWORK.md Research Positions respected -- no contradictions with established positions
- Open Questions from FRAMEWORK.md addressed if this chapter was expected to resolve them
- Progressive argument thread connects to prior chapters via summaries
- Reserved topics from STRUCTURE.md are not developed in this chapter

Category 4 - Formatting Norms Compliance (3-5 subchecks):
- Check thesis.json for norm (ABNT/APA), apply norm-specific citation style format rules
- LaTeX structure valid: \chapter{}, \section{}, \subsection{} hierarchy correct
- Labels use consistent convention: \label{chap:slug}, \label{sec:slug}
- Table and figure environments properly formed; cross-references use \ref{} correctly
- No raw special characters in prose (& % $ # _ escaped outside commands)
- Section lengths approximately match PLAN.md targets

**REVIEW.md output format (adapt GWD frontmatter):**
```yaml
---
type: chapter-review
chapter: NN
draft_path: "NN-01-DRAFT.tex"

re_review: false
previous_score: null
failed_checks:
  - category: "citation_validity"
    item: "Specific finding"
    evidence: "Quote from text"
    fix: "Suggested fix"
---
```

Use category identifiers: citation_validity, methodological_rigor, argumentative_coherence, formatting_norms.

**Preserve from GWD reviewer:**
- Adversarial stance and scoring methodology
- Re-verification mode (when re_review: true, focus on previously failed checks)
- Context assembly step using gtd-tools.js context --chapter N
- Reading PLAN.md for planned vs actual comparison
- Score threshold logic for pass/needs_revision/fail
