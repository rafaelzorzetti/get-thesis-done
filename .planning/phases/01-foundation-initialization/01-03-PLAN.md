  ```yaml
  description: Initialize a new thesis project through guided questioning and template population
  allowed-tools:

  ---
  ```
  - objective: Initialize a thesis project via deep questioning about topic, level, advisor, norms, language, and structure
  - execution_context: reference the workflow at `~/.claude/get-thesis-done/workflows/new-thesis.md` and all template files at `~/.claude/get-thesis-done/templates/`

  **get-thesis-done/workflows/new-thesis.md (full workflow):**
  Adapt from get-writing-done/workflows/new-book.md with these thesis-specific changes:

  **Step 1: Validate**
  - Check for existing canonical documents: FRAMEWORK.md, STYLE_GUIDE.md, STRUCTURE.md (not BIBLE.md, OUTLINE.md)
  - Check git initialization
  - Warn if overwriting

  **Step 2: Deep Questioning (THESIS-SPECIFIC)**
  Open with: "What is your thesis about?"

  Areas to explore (adapted from GWD for academic context):
  - **Research topic and question**: What is the central research question? What gap does it address?
  - **Thesis level**: Graduation (TCC), Master's (Dissertacao), or PhD (Tese)? This affects rigor, depth, and word count targets.
  - **Advisor**: Who is the advisor? Any co-advisor? What is the research group/lab?
  - **Institution**: University name, department/program, city
  - **Academic norms**: ABNT (default for Brazilian), APA, IEEE? Which specific standard?
  - **Language**: Primary language (PT-BR, EN, ES)? Secondary language for abstract?
  - **Methodology**: What research methods? Qualitative, quantitative, mixed? What theoretical framework?
  - **Structure**: How many chapters? What is the progression? (Introduction, Literature Review, Methodology, Results, Discussion, Conclusion is typical but varies)
  - **Key terms**: Technical terms that need precise definitions in the glossary
  - **Boundaries**: What is this thesis NOT about? What topics are explicitly excluded?
  - **Existing references**: Do they already have a .bib file from Zotero/Mendeley?
  - **Timeline**: Defense deadline? (affects target lengths and urgency)

  Readiness check (background): Can you fill:
  - FRAMEWORK.md: research question, at least 2 glossary terms, at least 1 methodological commitment
  - STYLE_GUIDE.md: academic voice register, citation style (ABNT/APA)
  - STRUCTURE.md: chapter structure with at least thesis and key points per chapter
  - main.tex: title, author, advisor, institution, language

  Decision gate (AskUserQuestion): "Ready to scaffold?" / "Keep exploring"

  **Step 3: Scaffold**
  Run `gtd-tools.js init --language {language} --level {level}`:
  ```bash
  node ~/.claude/get-thesis-done/bin/gtd-tools.js init --language {{language}} --level {{level}}
  ```
  Verify all files created.

  **Step 4: Populate Templates**
  CRITICAL: Populate ONE template at a time. Read scaffolded file, replace {{PLACEHOLDER}} values with author's answers, write back.

  Order:
  1. **4a: Populate FRAMEWORK.md** -- thesis statement, glossary terms with contrastive definitions, research positions, methodological commitments
  2. **4b: Populate STYLE_GUIDE.md** -- voice profile (academic register), citation style rules (ABNT/APA based on norm), language specifics
  3. **4c: Populate STRUCTURE.md** -- thesis-level arc, chapter structure with contracts, dependency map
  4. **4d: Populate main.tex** -- Replace all {{PLACEHOLDER}} values: title, author, advisor, institution, city, year, work type, preamble, language_babel. Generate \include{} lines for each chapter. Fill abstract placeholders with "[To be written]".
  5. **4e: Generate chapter stubs** -- For each chapter in structure, create `src/chapters/NN-slug.tex` from chapter.tex template with chapter title and section titles. Also create `.planning/chapters/NN-slug/` directories.
  6. **4f: Populate FIGURES.md** -- Set thesis title, leave catalog empty
  7. **4g: Create .planning/thesis.json** -- Already created by init, but verify it has correct level, language, norm, created date

  **Step 5: Verify Compilation (NEW -- not in GWD)**
  Attempt a test compilation to verify the scaffolded project compiles:
  ```bash
  cd src && node ~/.claude/get-thesis-done/bin/gtd-tools.js compile
  ```
  If compilation fails: diagnose and fix template issues. Common issues: missing LaTeX packages, encoding problems, babel language mismatch.
  If latexmk not installed: skip with warning "Install TeX Live to compile. See: https://tug.org/texlive/"

  **Step 6: Commit**
  ```bash
  git add .planning/FRAMEWORK.md .planning/STYLE_GUIDE.md .planning/STRUCTURE.md .planning/FIGURES.md .planning/thesis.json .planning/chapters/ src/
  git commit -m "docs: initialize thesis project with populated canonical documents and LaTeX structure"
  ```

  **Step 7: Report**
  ```
  Thesis project initialized:

    FRAMEWORK.md: Research question, glossary, positions populated
    STYLE_GUIDE.md: Academic voice profile and citation rules populated
    STRUCTURE.md: Structure with [N] chapters populated
    main.tex: LaTeX template configured for [INSTITUTION]
    [N] chapter stubs created in src/chapters/

    Thesis: [TITLE]
    Level: [graduation/master's/PhD]
    Language: [PT-BR/EN/ES]
    Norms: [ABNT/APA]

  Next steps:
    - Discuss a chapter: /gtd:discuss-chapter 1
    - Check progress: /gtd:progress
    - Compile PDF: /gtd:compile
  ```

  **Terminology section:**
  | Thesis Term | NOT This |
  |-------------|----------|
  | Thesis | Project |
  | Chapter | Phase |
  | Research question | Core value |
  | Advisor | Stakeholder |
  | Author | User |
  | Canonical documents | Planning files |

  **What NOT to do:**
  - Do NOT generate templates from scratch -- scaffold via CLI first, then populate
  - Do NOT load all templates at once -- read and populate one at a time
  - Do NOT fill sections the author did not discuss
  - Do NOT ask implementation/technical questions -- focus on thesis content
  - Do NOT use software jargon -- say "chapters" not "phases"
  - Do NOT rush questioning -- depth here saves time in every subsequent chapter
  - Do NOT skip the compilation verification step

  **Context budget:** ~35% (questioning ~15%, population ~15%, compile verification ~5%)

  <name>Task 2: Create /gtd:progress and /gtd:compile commands</name>
  <files>commands/gtd/progress.md, commands/gtd/compile.md</files>
  <action>
  Create `commands/gtd/` directory if it does not exist.

  **commands/gtd/progress.md (adapt from commands/gwd/progress.md):**
  ```yaml
  ---
  name: gtd:progress
  description: Show thesis progress dashboard with visual progress bar and next action
  allowed-tools:
    - Read
    - Bash
    - Glob
    - Grep
  ---
  ```

  Process (inline command, no workflow file needed):
  1. Run `node ~/.claude/get-thesis-done/bin/gtd-tools.js progress` to get chapter data (JSON)
  2. Read FRAMEWORK.md frontmatter for thesis title from `.planning/FRAMEWORK.md`
  3. Read `.planning/thesis.json` for thesis level and language
  4. Build enhanced dashboard:
     - **Header:** Thesis title, level, language
     - **Progress bar:** Visual bar `[========--] X/Y chapters (ZZ%)`
     - **Chapter table:** Ch | Title | Status | Coverage columns
       - Coverage: has SUMMARY.md and FRAMEWORK.md changelog entry (checkmarks)
     - **Current position:** First incomplete chapter and its status
     - **Next action recommendation:**
       - scaffolded -> `/gtd:plan-chapter N` or `/gtd:write-chapter N`
       - planned -> `/gtd:write-chapter N`
       - drafted -> `/gtd:review-chapter N`
       - reviewed -> continuity handled by review
       - all final -> "Thesis complete! Run /gtd:compile for final PDF."

  **commands/gtd/compile.md:**
  ```yaml
  ---
  name: gtd:compile
  description: Compile thesis LaTeX to PDF using latexmk
  allowed-tools:
    - Read
    - Bash
  ---
  ```

  Process (inline command):
  1. Run `node ~/.claude/get-thesis-done/bin/gtd-tools.js compile` from the project root
  2. Display results:
     - On success: "PDF compiled successfully: src/output/main.pdf"
     - On failure: Show extracted error messages with actionable suggestions
     - Common error patterns and fixes:
       - "Undefined control sequence" -> likely missing package
       - "Missing $ inserted" -> math mode issue
       - "File not found" -> missing \include{} target
       - "I found no \citation commands" -> biber needs .bib entries
  3. If `--clean` flag: run `gtd-tools.js compile --clean` first to clear auxiliary files
  </action>
