<task type="auto">
  <name>Task 2: Create gtd-tools.js CLI with init, progress, context, and compile commands</name>
  <files>get-thesis-done/bin/gtd-tools.js</files>
  <action>
  Create `get-thesis-done/bin/gtd-tools.js` by adapting gwd-tools.js with these changes:

  **Keep from GWD (copy and adapt):**
  - safeReadFile(), extractFrontmatter(), reconstructFrontmatter(), spliceFrontmatter() -- identical utility functions
  - execGit() -- identical
  - generateSlugInternal() -- identical
  - output() and error() -- identical
  - CLI router main() structure with --raw support

  **cmdInit (adapt from cmdBookInit):**
  - Template source: `get-thesis-done/templates/` (primary) or `.claude/get-thesis-done/templates/` (fallback)
  - Scaffold TWO directory trees:
    1. `.planning/` (agent workspace):
       - `.planning/FRAMEWORK.md` from template `framework.md`
       - `.planning/STYLE_GUIDE.md` from template `style-guide.md`
       - `.planning/STRUCTURE.md` from template `structure.md`
       - `.planning/chapters/` directory
       - `.planning/FIGURES.md` from template `figures.md` (empty catalog)
    2. `src/` (LaTeX source tree):
       - `src/main.tex` from template `main.tex`
       - `src/chapters/` directory
       - `src/references.bib` from template `references.bib` (empty with header comment)
       - `src/references/` directory (for PDFs)
       - `src/figures/` directory
  - Accept optional `--language` flag (default: pt-BR) and `--level` flag (default: master)
  - Store thesis config in `.planning/thesis.json`: `{ language, level, norm, created }`
  - Output JSON: `{ scaffolded, files_created, files_skipped, directories_created, thesis_config }`

  **cmdProgress (adapt from cmdBookProgress):**
  - Read from `.planning/chapters/` for chapter directories
  - Check for PLAN.md, DRAFT.tex (not DRAFT.md), REVIEW.md, SUMMARY.md
  - Check `src/chapters/` for final .tex files
  - Status hierarchy: scaffolded < planned < drafted < reviewed < final
  - Include thesis level from `.planning/thesis.json` in output
  - Table format: Ch | Name | Status (same as GWD but adapted labels)

  **cmdContext (adapt from cmdBookContext):**
  - Read `.planning/FRAMEWORK.md` (replaces BIBLE.md)
  - Read `.planning/STYLE_GUIDE.md` (same)
  - Read `.planning/STRUCTURE.md` (replaces OUTLINE.md)
  - Extract chapter entry from STRUCTURE.md using `#### Chapter NN:` pattern
  - Gather prior summaries from `.planning/chapters/` (same pattern)
  - Bundle sections: FRAMEWORK, STYLE GUIDE, CHAPTER CONTRACT, PRIOR CHAPTER SUMMARIES
  - Include thesis level and language from `.planning/thesis.json` in bundle header
  - Token estimate: word count * 1.3

  **cmdCompile (NEW -- not in GWD):**
  - Check `src/main.tex` exists, error if not
  - Check `latexmk` is installed via `which latexmk`, provide install hint if missing
  - Run: `latexmk -pdf -pdflatex="pdflatex -interaction=nonstopmode -halt-on-error" -bibtex-cond1 -outdir=src/output src/main.tex`
  - Use `-bibtex-cond1` which tells latexmk to run biber only when .bib files exist and are referenced
  - Capture stdout/stderr
  - Parse exit code: 0 = success, non-zero = failure
  - On success: report output PDF path (`src/output/main.pdf`)
  - On failure: extract error lines from log (lines containing "!" or "Error")
  - Output JSON: `{ success, pdf_path, errors[], warnings[] }`
  - Accept `--clean` flag to run `latexmk -C` before compilation

  **CLI Router:**
  - Commands: init, progress, context, compile
  - Keep framework (alias for future framework-keeper updates, like bible in GWD)
  - Keep summary (alias for future summary extraction, like summary in GWD)
  - Parse --chapter, --language, --level, --clean, --raw flags

  **Do NOT include:** bible or summary commands from GWD. These will be added in Phase 3 as framework update and summary extract. The router should have placeholder cases that error with "Coming in Phase 3" messages.

  Use `#!/usr/bin/env node` shebang. Zero npm dependencies -- only Node.js built-ins (fs, path, child_process).
  </action>
